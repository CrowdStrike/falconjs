## Project Overview

FalconJS (`crowdstrike-falcon` on npm) is a TypeScript SDK for the CrowdStrike Falcon API, supporting both browser and Node.js environments. It is **heavily auto-generated** from CrowdStrike's OpenAPI/Swagger specification using `openapitools/openapi-generator-cli` with the `typescript-fetch` generator.

## Common Commands

```bash
npm run build            # Compile TypeScript to dist/
npm run build:devel      # Compile including examples/
npm run build:browser    # Bundle for browser via esbuild
npm run lint             # Run ESLint on src/
npm run lint:fix         # Auto-fix lint issues
npm run format:fix       # Run Prettier on src/**/*.ts
npm run build:release    # Full release build: lint:fix → format:fix → clean → build → build:browser → npm pack
```

No test suite exists. The `npm test` command is a placeholder.

### Pre-PR Checklist

```bash
npm run format:fix
npm run lint:fix
npm run lint             # Ensure no remaining lint errors
npm run build
```

## Architecture

### Auto-Generated vs. Hand-Written Code

Most of the codebase is auto-generated. Auto-generated files contain the header `This class is auto generated by OpenAPI Generator`. **Do not manually edit auto-generated files** — changes will be overwritten on the next regeneration.

**Auto-generated:**

- `src/apis/` — API client classes (~123 files)
- `src/models/` — Request/response TypeScript interfaces (~2,650 files)
- `src/runtime.ts` — Base classes (`BaseAPI`, `Configuration`), middleware system, request handling

**Hand-written (safe to edit):**

- `src/client.ts` — `FalconClient` class, the primary typed entry point for end-users
- `src/middleware/` — OAuth2 token management (`oauth2.ts`) and UserAgent (`useragent.ts`) middleware
- `src/error.ts` — `FalconError` class and `FalconErrorExplain()` helper
- `src/event_stream.ts` — Streaming API support
- `src/FalconCloud.ts` — Cloud region definitions (us-1, us-2, eu-1, us-gov-1)
- `src/index.ts` — Main entry point (re-exports all modules)

### Client Pattern

**FalconClient** (`src/client.ts`) — Strongly-typed API clients grouped by service (e.g., `client.sensorDownload`, `client.detects`). Uses the generated API classes from `src/apis/`.

> **Note:** A dynamic uber class (`Falcon`) with a `command()` method previously existed but was reverted in PR #311. Only `FalconClient` is available.

### Code Regeneration

Run `./rebuild.sh` to regenerate from an updated Swagger spec. Requires Docker or Podman. The process:

1. Place `swagger.json` in `specs/` (manual download)
2. The script applies JQ (`specs/transformation.jq`) and sed (`specs/transformation.sed`) transforms
3. Generates TypeScript via OpenAPI Generator in a container
4. Replaces auto-generated files in `src/`, preserving hand-written files
5. Runs format, lint, and build

## Code Style

- Prettier: 200 char print width, 4-space indentation (spaces, not tabs; 2-space for YAML)
- ESLint: flat config (`eslint.config.mjs`), TypeScript recommended rules; unused vars allowed if prefixed with `_`
- `@typescript-eslint/no-explicit-any` is disabled for `falcon.ts`, `endpoints.ts`, and `_endpoints/*.ts` (these files were reverted in PR #311 but the ESLint rule remains)
- Target: ES2016, CommonJS modules, strict mode enabled

## Releasing

To prepare a release, use the `/release` skill:

```
/release patch   # bug fixes and maintenance
/release minor   # new features (post-1.0) or breaking changes (pre-1.0)
/release major   # breaking changes (post-1.0)
```

This automates version bumping across `rebuild.sh`, `src/middleware/useragent.ts`, `package.json`, and `package-lock.json`, runs the pre-PR checklist, and opens a release PR. See `docs/devel.md` for the manual process.
